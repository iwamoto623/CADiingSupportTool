# CADEditTool 設計書

## 概要

CADツールの前面に透明オーバーレイウィンドウを表示し、楕円を描画するCADサポートアプリ。
背景のCADツールは引き続き操作可能（クリックスルー対応）。

---

## 要件

| 項目 | 内容 |
|------|------|
| プラットフォーム | Windows |
| 常時最前面 | 常にCADツールの上に表示 |
| 背景透過 | ウィンドウ自体は透明 |
| クリックスルー | 背景透過モード時はすべてのクリックがCADツールに届く |
| 描画機能 | 楕円の描画・編集・削除 |

---

## アーキテクチャ

```
CADEditTool (透明オーバーレイ)
│
├── UI層 (overlay_window.py)
│   └── 透明ウィンドウ + 図形描画キャンバス
│
├── 図形層 (shapes/)
│   ├── base_shape.py     # 図形の基底クラス
│   └── ellipse.py        # 楕円クラス（BaseShape を継承）
│
├── 操作ウィンドウ (control_window.py)
│   ├── ツールバー（モード切替・消去ボタン）
│   └── 画面内を自由にドラッグ移動できるフローティングウィンドウ
│
├── パラメータウィンドウ (param_window.py)
│   └── 図形選択時に個別に開くプロパティ編集ウィンドウ（同じくフローティング）
│
├── 図形エディタ (shape_editor.py)
│   └── 選択中の図形にハンドルを表示・ドラッグで直接編集
│
├── 入力層 (input_handler.py)
│   └── マウスイベント処理（ハンドル優先 → 図形選択 → 新規作成）
│
├── window_tracker.py  ※オプション
│   └── 対象ウィンドウへのオーバーレイ追従（本体と独立）
│
└── main.py
    └── アプリ起動・ウィンドウ制御
```

| 用途 | ライブラリ |
|------|-----------|

### オーバーレイの2ウィンドウ構成

`-transparentcolor` (Win32: `LWA_COLORKEY`) の制約により、透明色に設定したピクセルはモード関係なく常にクリックも透過してしまう。そのため、**2ウィンドウ構成**を採用する。

```
z-order（上から）
  ┌─────────────────────────┐
  │ 操作ウィンドウ (control) │ ← 常に最前面（200ms ごとに lift）
  ├─────────────────────────┤
  │ click_win               │ ← 1%不透明。描画モードでクリック受取
  │ （フルスクリーン）        │   背景透過モードで WS_EX_TRANSPARENT 付与
  ├─────────────────────────┤
  │ visual_win              │ ← 図形描画。常時 WS_EX_TRANSPARENT
  │ （透明背景・クリックスルー）│   -transparentcolor '#000000'
  ├─────────────────────────┤
  │ CADツール               │
  └─────────────────────────┘
```

| ウィンドウ | 役割 | 透過設定 |
|-----------|------|---------|
| `visual_win` | 図形の描画・表示 | `WS_EX_TRANSPARENT` 常時付与、背景 `#000000` |
| `click_win` | マウスイベント捕捉 | 描画モード: 解除 / 背景透過モード: 付与 |

Win32 API の拡張ウィンドウスタイル：
- `WS_EX_LAYERED` : 透明ウィンドウを有効化
- `WS_EX_TRANSPARENT` : クリックを背面ウィンドウに透過

## 画面仕様

- オーバーレイ：フルスクリーン、または「描画範囲の選択オプション」でCADウィンドウに合わせたサイズ
- 背景：完全透明
- 描画した楕円のみ表示（枠線あり、塗りつぶしなし or 半透明）
- 操作ウィンドウ：オーバーレイとは独立したフローティングウィンドウ（常時最前面）
  - ドラッグで画面内を自由に移動可能
  - クリックスルーの影響を受けない
  - モード切替ボタン・消去ボタンを配置

---

## 操作仕様

| 操作 | 動作 |
|------|------|
| ドラッグ（空領域） | 楕円を新規描画 |
| 図形クリック | 選択 → ハンドル表示 + パラメータウィンドウが開く |
| ハンドルドラッグ | 図形を直接変形（下表参照） |
| 空領域クリック | 選択解除 |
| Delete キー | 選択中の図形を削除 |
| Esc キー | 選択解除 |
| モード切替ボタン | 描画モード ↔ 背景透過モードをトグル |

### ハンドル一覧

```
          ○ 回転ハンドル（紫）
          │
    ──────●──────  ← 長軸ハンドル（緑）
          │
    ○ 短軸ハンドル（橙）

    ● 中心ハンドル（青）: 移動
```

| ハンドル | 色 | 操作 |
|---------|-----|------|
| 中心 ● | 青 | 図形全体を移動 |
| 長軸端 ● | 緑 | 長軸半径（radius_x）をリサイズ |
| 短軸端 ● | 橙 | 短軸半径（radius_y）をリサイズ |
| 回転 ○ | 紫 | 回転角度を変更（中心からの角度） |

## 図形クラス設計

### 基底クラス `BaseShape`

すべての図形が共通で持つインターフェース：

```python
class BaseShape:
    # 共通パラメータ
    x: float          # 中心X座標
    y: float          # 中心Y座標
    rotation: float   # 回転角度（度）
    color: str        # 枠線の色
    line_width: int   # 線の太さ
    visible: bool     # 表示/非表示

    def draw(self, canvas): ...       # キャンバスへの描画
    def contains(self, x, y): ...    # 点が図形内かどうか
    def get_params(self): ...        # パラメータをdict形式で返す
    def set_params(self, params): ...# パラメータをdictで一括更新
```

### 楕円クラス `Ellipse(BaseShape)`

```python
class Ellipse(BaseShape):
    radius_x: float   # 長軸の半径
    radius_y: float   # 短軸の半径
```

## パラメータウィンドウ

- 描画モードで図形をクリック選択すると**個別に独立したウィンドウ**が開く
- 複数図形を選択すれば複数ウィンドウが同時に開く
- 各パラメータは入力フィールドで直接編集し、リアルタイムでオーバーレイに反映
- ウィンドウを閉じると選択解除

```
┌─────────────────────────┐
│ 楕円 #1                  │
├─────────────────────────┤
│ X座標:    [ 320.0 ]      │
│ Y座標:    [ 240.0 ]      │
│ 長軸半径: [ 100.0 ]      │
│ 短軸半径: [  60.0 ]      │
│ 回転角度: [  45.0 ] °    │
│ 線の色:   [  赤   ]      │
│ 線の太さ: [   2   ]      │
└─────────────────────────┘
```

## 描画範囲の選択オプション（window_tracker.py）

本体の処理とは独立したオプションモジュール。
有効にするとオーバーレイを特定ウィンドウに追従させる。無効時はフルスクリーン表示のまま動作する。

### 概要

```
起動時
 ↓
[描画範囲の選択オプション]
 ├── フルスクリーン（デフォルト）→ そのまま起動
 └── ウィンドウを選択 → 対象HWND を取得 → window_tracker 起動
                                               ↓
                                        タイマー(100ms)で追従
```

### 処理内容

| 処理 | 内容 |
|------|------|
| ウィンドウ一覧取得 | `EnumWindows` で実行中の可視ウィンドウを列挙 |
| サイズ・位置取得 | `GetWindowRect` で対象ウィンドウの矩形を取得 |
| オーバーレイ追従 | `SetWindowPos` でオーバーレイを同位置・同サイズに更新 |
| 最小化検出 | `IsIconic` で最小化を検出 → オーバーレイを非表示 |
| 終了検出 | `IsWindow` で対象ウィンドウの生存確認 → 消えたら追従停止 |

### 本体との分離方針

- `overlay_window.py` は「矩形で初期化される」だけで、追従ロジックは一切持たない
- `window_tracker.py` が外部からオーバーレイのサイズ・位置を更新するだけ
- オプションを使わない場合は `window_tracker.py` は一切呼ばれない
